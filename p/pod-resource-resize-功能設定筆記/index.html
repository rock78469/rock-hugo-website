Pod resource resize 功能設定筆記

Dec 21, 2023
閱讀時間: 1 分鐘

kubernetes v1.27 的新功能 連結

變更pod resource的 request 或 limit 時，不須重啟Pod**

測試環境

kind cluster v1.27

開始之前

因v1.27是alpha階段，測試之前須先開啟 featureGates InPlacePodVerticalScaling 功能

否則yaml怎麼加都沒有效果

featureGates.InPlacePodVerticalScaling=true

設定方式

在containers 底下新增resizePolicy

spec:
  containers:
  - name: example-pod
    resizePolicy:
    - resourceName: cpu
      restartPolicy: NotRequired
    - resourceName: memory
      restartPolicy: RestartContainer

範例yaml，即是修改cpu時，pod不會重啟，但修改memory時，會重啟pod

apiVersion: v1
kind: Pod
metadata:
  name: nginx-demo
  namespace: default
spec:
  containers:
  - name: nginx-demo
    image: nginx
    resizePolicy:
    - resourceName: cpu
      restartPolicy: NotRequired
    - resourceName: memory
      restartPolicy: RestartContainer
    resources:
      limits:
        memory: "200Mi"
        cpu: "200m"
      requests:
        memory: "100Mi"
        cpu: "100m"

問題

但目前還無法調整resource去變更 pod的QoS

例如想要變更Qos Type Guaranteed to Burstable 或者反過來，都會遇到錯誤

1
2
3
4

# Burstable to Guaranteed
The Pod "nginx-demo" is invalid: metadata: Invalid value: "Guaranteed": Pod QoS is immutable
# Guaranteed to Burstable
The Pod "nginx-demo" is invalid: metadata: Invalid value: "Burstable": Pod QoS is immutable

補充：

Pod QoS

  • Guaranteed：Pod 裡的每个Container都必須有Memory/CPU Limit或Request，而且值必須相等。
  • Burstable：Pod 裡至少有一個Container有Memory或者 CPU 請求且不满足 Guarantee 等級的要求，即Memory/CPU 的值設置的不同。
  • BestEffort：Pod 內没有任何Memory或者 CPU 的Limit或Request。
